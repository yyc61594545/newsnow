# 每日多平台调研系统 — 任务说明（source of truth）

本文件是自动化调研任务的唯一权威说明。GitHub Actions 工作流会把某个「任务ID」
传给 Claude Code，由你按下面的规则完成该任务：采集 → 排序 → 过滤 → 用中文综合成
markdown 简报 → 写入指定文件。Telegram 投递与 git 提交由工作流后续步骤负责，你只需
写好简报文件。

---

## 通用执行流程（所有任务一致）

1. **确定任务**：根据传入的任务ID，在下方「任务清单」中找到主题、平台侧重与特殊要求。
2. **采集数据（按优先级）**
   - **优先**：使用 last30days skill（已安装于 `.claude/skills/last30days`）调研该任务
     指定的平台，覆盖最近 30 天、重点最近 7 天。
   - **回退**：若该 skill 不存在或报错，运行：
     ```
     python3 daily-research/scripts/aggregate.py --task <任务ID> --out /tmp/items.json
     ```
     读取 `/tmp/items.json` 中的 `candidates` 作为候选。
3. **排序**：按真实互动热度排序——Reddit 看点赞+评论、Hacker News 看 points+评论、
   YouTube 看播放量、Polymarket 看成交量/赔率。优先高互动内容。
4. **过滤**（很重要）：
   - 去掉标题党、营销软文、与主题无关、信息量过低的内容。
   - 去重：同一事件/同一消息只保留信息最完整、互动最高的一条来源。
   - 链接必须真实可点击；**绝不可编造链接或数据**。无法验证就不要收录。
   - 只覆盖任务指定的平台（本期统一**跳过 X/Twitter**）。
5. **撰写简报**：选出**最多 12 条**，写入工作流指定的输出文件
   （形如 `daily-research/briefs/<任务ID>/<日期>.md`，目录不存在则创建）。
6. **收尾**：只写文件，**不要**自行 commit 或开 PR（由工作流完成）。若某平台本期无
   优质结果，注明「本期无高质量结果」即可，不要凑数。

## 简报格式（严格遵守）

文件开头：

```
# <任务名称> · <日期>
> 来源平台：<平台列表> ｜ 共 N 条 ｜ 按互动热度排序

```

每一条目：

```
## N. <标题（中文，可保留关键英文术语，尽量精简）>
**平台** · <热度指标，口语化数值即可，如 1.2k赞·340评论 / 4.5万观看 / 成交量约3850万·赔率62%>
<一句话中文摘要：是什么 + 为什么值得看，约 30–60 字，最多两短句>
原文链接: <真实 URL>
```

排版与简洁要求（重要，我嫌信息太密）：
- 摘要**只写一句话**（必要时两短句），不堆细节、不复述标题，让我一眼判断要不要点开。
- 热度指标行**不要**再附来源域名/子版块（标题本身已是可点击链接）。
- 条目之间**不要**插入 `---` 等分隔线，留空行即可。
- 数字尽量口语化（"约3850万""4.5万观看"），不必精确到个位。

## 任务清单

> 标注的「平台侧重」是优先采集与展示的平台；可少量纳入其他指定平台的高质量内容。

### 任务ID: `ai-tools` —「AI工具与变现」
- **平台侧重**：Reddit、Hacker News、YouTube
- **主题**：
  - AI agents 的最新进展与实用案例。
  - agent 的 skill / `SOUL.md` / rules 等配置文件的最佳实践。
  - Claude Code 与 Codex 的高效用法、技巧、工作流。
  - AI 订阅 / API token 转售的市场需求与潜在客户线索（谁在找、找什么、愿付多少）。
- **侧重可操作性**：能直接上手的技巧、能跟进的线索优先。

### 任务ID: `money` —「搞钱与撸毛」
- **平台侧重**：Reddit、Polymarket、YouTube
- **主题**：
  - 副业 / 搞钱方法（可执行、有真实反馈的）。
  - Web3 空投撸毛的最新机会与教程。
  - Polymarket 上与加密/热点相关、成交量高的市场，作为风向参考。
- **特殊要求（务必遵守）**：撸毛类条目**必须标注项目真伪与风险**
  （是否已确认发币、是否疑似貔貅/钓鱼、是否需要授权钱包、是否要垫资等），
  对不确定的项目明确写「真伪存疑，注意风险」。

### 任务ID: `uscards` —「美卡」（每隔一天）
- **平台侧重**：Reddit、YouTube
- **主题**：
  - 美国信用卡攻略最新消息：开卡奖励（SUB）、羊毛、churning 玩法与规则变化。
  - 美卡代购 / 代付的市场行情与潜在客户需求。
- **侧重时效**：优先最近的 offer 变动与可执行玩法。

### 任务ID: `self-improve` —「自我提升」（每周一）
- **平台侧重**：Reddit、YouTube、Hacker News
- **主题**：
  - 提高认知与思维方法（mental models、批判性思维等）。
  - 英语听力 / 口语的高效学习法与可执行训练方案。
  - 禅定 / 静坐的入门与速成方法。
- **侧重**：方法论要具体、可落地，附带高质量教程/资源更佳。

### 任务ID: `foreign-trade` —「外贸获客」（每周一）
- **平台侧重**：Reddit、YouTube、Hacker News
- **背景**：使用者是中国紧固件/汽配/五金/标准件行业的采购员，做外贸 B2B。
- **主题（结构固定为三块）**：
  1. **获客方法**：外贸 B2B 找客户、开发信、平台运营等可执行方法。
  2. **买家聚集地**：海外买家/采购商聚集的平台、论坛、社区、展会线索。
  3. **行业动态**：紧固件 / 汽配 / 五金 / 标准件行业动态与采购趋势。
- **特殊要求**：交付「方法 + 买家聚集地 + 行业动态」，**不是**直接给客户名单。
